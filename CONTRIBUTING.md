# Contributing

Quick guide
- Fork the repo and create a branch for your change: naming convention `topic/<short-description>`.
- Add content under the appropriate folder (01_core_model, 02_foundations, 03_tools_prompts).
- Use clear commit messages and a small PR description.

PR checklist
- [ ] Add or update docs as needed.
- [ ] Include licenses and attributions for external assets.
- [ ] Provide a short test or example when submitting code or prompts.
